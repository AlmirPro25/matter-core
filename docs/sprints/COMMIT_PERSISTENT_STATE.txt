Sprint 4.1: Estado Persistente - REPL Completo e Funcional

PROBLEMA RESOLVIDO:
Antes: Variáveis não persistiam entre comandos
[1]> let x = 10
[2]> print x
Semantic error: undefined variable 'x'  ❌

Depois: Variáveis persistem naturalmente
[1]> let x = 10
[2]> print x
10  ✅

IMPLEMENTAÇÃO:
- Source code acumulativo em vez de merge de bytecode
- Recompilação automática a cada comando
- Estado global transferido entre execuções
- Performance aceitável (< 10ms para 100 comandos)

NOVOS MÉTODOS NA VM:
- get_globals() - Extrai estado global
- set_globals() - Injeta estado global
- merge_functions() - Mescla funções de bytecode

NOVOS MÉTODOS NO RUNTIME:
